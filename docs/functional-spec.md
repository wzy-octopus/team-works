# 機能定義書

## サービス概要

チームの各メンバーのタスクを見える化し、AI時代の新しい協同を実現するチームタスク管理サービス。

---

## 1. タスク入力

### 入力チャネル
- **WEBアプリ**：ブラウザから直接入力
- **生成AI連携**：ChatGPT / Claude Desktop などから自然言語で入力
- **チャットシステム連携**：Slack / Microsoft Teams のBotを通じて入力

### 入力内容
| フィールド | 必須 | 説明 |
|---|---|---|
| タスク名 | ○ | 作業内容の名称 |
| 予定時間 | ○ | 作業にかかる見込み時間（時間単位。例：1.5h） |
| プロジェクト | ○ | 紐づけるプロジェクトを選択 |
| 非表示フラグ | - | チームメンバーに見せたくない場合に設定 |

### 入力フォーマット（AI・チャット連携時）
自然言語を解析し、タスク名と予定時間を自動抽出してシステムへ書き込む。  
例：「今日は要件定義2時間、コードレビュー1時間やります」

---

## 2. タスク管理

### タスク一覧・可視化
- 自プロジェクトのメンバー全員のタスクを一覧表示
- 日付（当日）ベースのビューで表示
- タスクの進行ステータスを管理

### ステータス管理
タスクは以下の3段階で管理し、順に遷移する。

| ステータス | 説明 |
|---|---|
| 未着手 | 初期状態 |
| 進行中 | 作業開始後の状態 |
| 完了 | 作業完了 |

WEBアプリ上ではチェックボタンのクリックで 未着手 → 進行中 → 完了 → 未着手 とサイクルする。

### 非表示機能
- タスク単位で「非表示」に設定可能
- 非表示タスクは本人のみが閲覧できる
- プロジェクトメンバーおよび上長からも見えない（存在自体を表示しない）
- 本人のダッシュボードカードには「🔒 非表示 N件」としてカウントのみ表示

---

## 3. 週間業務報告

### 自動サマリ生成
- 毎週末（金曜日EOD または 土曜日AM）にAIが自動実行
- 対象：その週に本人が登録・完了したタスクの履歴（非表示タスクを除く）
- AIが作業内容・時間・完了状況を集約し、サマリ文を生成
- 生成後は本人が内容を確認し、任意で再生成できる

### 報告書の完成フロー
1. AIがサマリを自動生成し、本人へ通知（Slack / Teams Bot またはメール）
2. 本人が以下を追記
   - 今週の所感（必須）
   - 疑問・気になった点（任意）
   - 課題・改善提案（任意）
3. 提出ボタン（または送信コマンド）で上長へ連携

### 上長による閲覧・フィードバック
- WEBアプリ上で担当メンバー全員の週報を受信トレイ形式で一覧・詳細表示
- 生成AI連携経由で「〇〇さんの今週の週報を見せて」と照会可能
- Slack / Teams のBot経由でも閲覧可能
- 上長はWEBアプリ上で以下のフィードバックを送ることができる
  - テキストコメント（任意）
  - リアクション：👍 よくできました / ⭐ 今週のベスト / ❤️ ありがとう / 🎉 おめでとう / 💪 引き続き頑張れ / 💡 参考になりました
  - テキストとリアクションはどちらか一方のみでも送信可能

### 週報のステータス
| ステータス | 説明 |
|---|---|
| 下書き | 作成中・未提出 |
| 提出可能 | 必須項目（所感）が入力済み |
| 提出済み | 上長への送付完了 |
| FB済み | 上長がフィードバックを送信済み |

---

## 4. 通知

WEBアプリ内の通知センターは設けない。通知はすべて外部チャネル経由で行う。

| イベント | 通知手段 |
|---|---|
| 週報AI生成完了 | Slack / Teams Bot またはメール |
| 週報提出完了 | 画面上の即時フィードバック（トースト表示） |
| 上長からのフィードバック受信 | Slack / Teams Bot またはメール |

---

## 5. マルチテナント

- テナント（企業・組織）単位でデータを完全分離
- ユーザーは複数テナントに所属可能。ログイン時にテナントを選択して入室する
- テナント管理者がユーザー・プロジェクト・上長設定を管理
- テナント間のデータ参照は不可

---

## 6. マルチプロジェクト

### プロジェクト管理
- テナント内に複数のプロジェクトを作成可能
- メンバーは複数のプロジェクトに同時参加可能

### プロジェクト間のデータ分離
- タスクはプロジェクト単位で管理
- 異なるプロジェクト間でのタスク情報の横断参照は不可
- 週報は人単位で作成されるが、プロジェクトフィルタで絞り込んで集計できる

### ロール

#### テナントロール（システム権限）
| ロール | 説明 |
|---|---|
| テナント管理者 | ユーザー・プロジェクト・ロールの管理。全権限を持つ |
| 上長 | 担当メンバーのタスク閲覧・週報受け取り・フィードバック送信 |
| メンバー | タスクの登録・管理・週報作成・提出 |

#### プロジェクト内ロール
| ロール | 説明 |
|---|---|
| リード | プロジェクトの進行管理担当 |
| メンバー | 一般参加者 |

---

## 7. 外部連携インターフェース

### 生成AI連携（ChatGPT / Claude Desktop など）
- APIエンドポイント（MCP または REST）を通じてタスクの登録・照会を実現
- 自然言語でのタスク入力・完了操作・週報確認をサポート

### チャットシステム連携（Slack / Microsoft Teams）
- Bot（Webhook / OAuth）によりチャンネルまたはDMからタスク操作
- 通知配信：週報生成通知、フィードバック受信通知

---

## 8. 認証・ログイン

- Google OAuth / Microsoft OAuth によるSSO認証
- メールアドレス＋パスワードによる認証も提供
- ログインは2ステップ：①認証 → ②テナント選択
- 複数テナントに所属するユーザーはログイン後に入室するテナントを選択する

---

## 9. 非機能要件（概要）

| 項目 | 方針 |
|---|---|
| 認証 | SSO（Google / Microsoft OAuth）＋メール・パスワード認証 |
| データ保存 | タスク・週報データはテナント・プロジェクト単位で論理分離 |
| API | REST API または MCP サーバーとして外部連携を提供 |
