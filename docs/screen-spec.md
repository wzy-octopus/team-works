# 画面定義書

## 概要

本書はデザインプロトタイプ（`design/` フォルダ内のHTMLファイル）をもとに、各画面の目的・構成要素・インタラクションを定義する。

---

## 画面一覧

| 画面名 | ファイル | 対象ロール |
|---|---|---|
| ログイン | login.html | 全ロール |
| チームダッシュボード | dashboard.html | メンバー・上長 |
| マイタスク | mytasks.html | メンバー |
| 週間レポート（作成） | weekly-report.html | メンバー |
| 週報受信トレイ | manager-inbox.html | 上長 |
| 週報詳細・フィードバック | manager-report-detail.html | 上長 |
| 管理設定 | admin.html | テナント管理者 |

---

## 1. ログイン画面 `login.html`

### 目的
サービスへの入口。認証とテナント選択を2ステップで行う。

### レイアウト
左右分割レイアウト。左：ブランディング、右：フォーム。

### 左パネル（ブランディング）
- サービスロゴ・名称・キャッチコピー
- 主要機能の紹介（タスクの透明化 / AI週報 / Slack・Teams連携）

### 右パネル：STEP 1 認証
| 要素 | 内容 |
|---|---|
| SSOボタン | Google アカウントでログイン / Microsoft アカウントでログイン |
| メール入力 | メールアドレス |
| パスワード入力 | パスワード（表示/非表示切替あり） |
| オプション | ログイン状態を保持（チェックボックス）/ パスワードを忘れた |
| エラー表示 | 認証失敗時にインライン表示 |

### 右パネル：STEP 2 テナント選択
- 所属テナントをカード形式で選択
- 複数テナント所属ユーザーは入室先を選ぶ
- 選択後「ワークスペースに入る」でダッシュボードへ遷移

### ステップインジケーター
認証 → テナント選択 の2ステップをヘッダー部分に表示。完了済みステップは ✓ に変化。

## 2. チームダッシュボード `dashboard.html`

### 目的
プロジェクトチーム全員の当日タスク状況を俯瞰する。

### ヘッダー
- 表示中プロジェクト名・日付
- ビュー切替トグル（メンバー別 / カンバン）
- タスク追加ボタン
- ダーク/ライト切替

### サマリカード（4枚）
本日のタスク総数 / 完了数 / 進行中数 / 未着手数

### メンバー別ビュー
メンバーごとのカードを格子表示。各カードに：
- メンバー名・役割・アバター
- 完了率（%）と進捗バー
- タスク一覧（タスク名・予定時間・ステータスバッジ）
- 非表示タスクがある場合は「🔒 非表示 N件」と表示（存在は示すが内容は非表示）
- 作業時間サマリ（完了時間 / 予定時間合計）

### カンバンビュー
未着手 / 進行中 / 完了 の3列。各カードに：
- タスク名・カテゴリチップ（開発/企画/デザイン/MTG/QA/運用）
- 担当者アバター・名前
- 予定時間

## 3. マイタスク `mytasks.html`

### 目的
自分のタスクの登録・管理・進捗確認を行う。

### サマリカード（4枚）
本日のタスク数 / 完了数 / 予定時間合計 / 非表示タスク数

### タスク入力エリア
| 要素 | 内容 |
|---|---|
| タスク名 | テキスト入力（Enter で追加） |
| 予定時間 | テキスト入力（例：1.5h） |
| プロジェクト | プルダウン選択 |
| 非表示トグル | ON/OFFスイッチ |
| 追加ボタン | クリックまたはEnterで登録 |
| AI連携ヒント | 「チャットから自然言語で登録できます」のバナー |

### フィルタバー
すべて / 未着手 / 進行中 / 完了 のタブ切替。件数バッジ付き。  
並び替え（登録順 / ステータス順 / 時間順）のセレクト。  
全体の進捗バー（完了N / 全体N）。

### タスク一覧
ステータスでグループ化して表示（進行中→未着手→完了の順）。  
各タスク行に：
- チェックボタン（クリックで 未着手→進行中→完了→未着手 とサイクル）
- タスク名・プロジェクトバッジ・登録時刻
- 予定時間・ステータスバッジ
- 非表示アイコン（👁/🔒 クリックで切替）
- 編集・削除アクションボタン（ホバー時に表示）

### 今週の作業時間グラフ
月〜金の曜日別バーチャート。今日の作業時間をハイライト。  
「週報を作成 →」ボタンで週間レポート画面へ遷移。

## 4. 週間レポート（メンバー） `weekly-report.html`

### 目的
AIが自動生成した週次サマリに所感・疑問・課題を追記し、上長へ提出する。

### ヘッダー
- 週報のステータスピル（下書き / 提出可能 / 提出済み）
- ダーク/ライト切替

### 週ナビゲーター
‹ › で週を移動。プロジェクトフィルタタブで対象プロジェクトを絞り込み。

### AIサマリカード（左カラム）
- 完了率リング・完了件数・作業時間 のサマリ統計
- AI生成テキスト（週の作業内容の要約文）
- 完了タスク一覧（タスク名・プロジェクト・時間）
- 「再生成」ボタンでAIサマリを再取得可能

### 所感追記エリア（左カラム）
| フィールド | 必須 | 内容 |
|---|---|---|
| 今週の所感 | ○ | 自由記述。文字数カウント付き |
| 疑問・気になった点 | - | 確認事項・判断できなかった点 |
| 課題・改善提案 | - | 来週以降への提言 |

### 提出サイドバー（右カラム）
- 送付先上長の表示
- 提出チェックリスト（AIサマリ生成済み / 各フィールドの入力状況をリアルタイム反映）
- 「週報を提出する」ボタン（チェックリスト全完了でアクティブ化）
- 「下書きとして保存」ボタン
- 提出完了時にモーダルで完了通知

### 過去の週報（右カラム）
過去の週報を提出済み / 下書きのステータスつきで一覧表示。

---

## 5. 週報受信トレイ `manager-inbox.html`

### 目的
上長が担当メンバーから届いた週報を一覧で確認する。

### ヘッダー
- 担当メンバー数
- ダーク/ライト切替

### サマリカード（4枚）
未読数 / 提出済み数 / 未提出数 / フィードバック済み数

### ツールバー
- フィルタタブ：すべて / 未読 / 提出済み / 未提出
- 週ナビゲーター（‹ › で週移動）
- 並び替えセレクト

### 週報リスト
各メンバーの週報を1行で表示：
- 未読は左ボーダー強調・メンバー名を太字
- メンバーのアバター・名前・役割
- プロジェクトバッジ・週報プレビューテキスト
- 提出日時・ステータスピル（未読 / FB済み）
- 完了件数・作業時間・完了率
- 未提出メンバーは点線ボーダー・グレーアウトで表示

## 6. 週報詳細・フィードバック `manager-report-detail.html`

### 目的
週報の内容を詳細確認し、フィードバックを返す。

### ヘッダー
- 「← 受信トレイ」戻るボタン
- メンバー情報（アバター・名前・役割・プロジェクト・対象週）
- 週報ステータスピル（未読 / 既読 / FB済み）

### 週サマリ帯
完了件数・作業時間・完了率を横並びで表示。

### AIサマリカード（左カラム）
- 統計（完了件数・時間・完了率）
- AI生成テキスト（読み取り専用）
- 完了タスク一覧

### メンバー所感カード（左カラム）
メンバーが記入した所感・疑問・課題を読み取り専用で表示。  
疑問・気になった点に「要確認」タグが付いている場合は警告バナーで強調。

### フィードバックカード（右カラム）
**リアクション（6種）**
| リアクション | 意味 |
|---|---|
| 👍 | よくできました |
| ⭐ | 今週のベスト |
| ❤️ | ありがとう |
| 🎉 | おめでとう |
| 💪 | 引き続き頑張れ |
| 💡 | 参考になりました |

- リアクションはトグル式（クリックでON/OFF・カウント表示）
- テキストコメント欄（任意）
- リアクションとコメントはいずれか一方のみでも送信可能
- 送信後、リアクションは送信済みチップとして表示固定
- 「既読にする」ボタン（コメントなしで既読にする場合）

### 週報履歴（右カラム）
- 過去5週分の完了率トレンドバーチャート
- 週次リスト（週・完了件数・ステータス・完了率）

## 7. 管理設定 `admin.html`

### 目的
テナント管理者がメンバー・プロジェクト・ロール・上長設定を管理する。

### タブ構成
3タブで切り替え。

---

### タブ1：メンバー管理
**ツールバー**：名前・メール検索、ロールフィルタ、プロジェクトフィルタ、招待ボタン

**メンバーテーブル**

| 列 | 内容 |
|---|---|
| メンバー | アバター・名前・メールアドレス |
| ロール | インラインセレクトで変更可能（管理者/上長/メンバー） |
| 参加プロジェクト | プロジェクトバッジ一覧 |
| ステータス | アクティブ / 離席中 / オフライン |
| 最終ログイン | 日時 |
| アクション | 編集・削除 |

**招待バナー**：メールアドレス入力 + ロール選択 + 招待メール送信

---

### タブ2：プロジェクト管理
プロジェクトカードを格子表示。各カードに：
- プロジェクト名・カラー・参加人数
- 参加メンバー一覧（プロジェクト内ロール変更・除外が可能）
- メンバー追加セレクト
- 編集・削除ボタン

「新規プロジェクト作成」カード（点線）をクリックでモーダルを表示。  
モーダルでプロジェクト名・カラー・説明を設定。

---

### タブ3：ロール・上長設定
**報告ライン図**  
メンバー → 上長 の週報送付ラインをフロー図で可視化。未設定の上長は「?」で表示。

**ロール設定テーブル**

| 列 | 内容 |
|---|---|
| メンバー | アバター・名前 |
| システムロール | テナントロールのバッジ表示 |
| 週報の送付先（上長） | セレクトで上長を指定 |
| 権限 | メンバー管理 / プロジェクト管理 / 週報閲覧 のON/OFFトグル |
| 保存 | 💾ボタンで個別保存 |

---

## 共通仕様

### サイドバー
全画面共通のナビゲーション。

**メンバー・上長向け**
- ダッシュボード / マイタスク / 週間レポート / ⚙️ 管理設定
- 参加プロジェクト一覧（カラードット付き）
- ログインユーザー情報（アバター・名前・役割）

**上長向け（上長ビュー時）**
- チームダッシュボード / 週報受信トレイ / ⚙️ 管理設定 / レポート分析
- 「上長ビュー」ロールチップを表示

**管理者向け**
- 「テナント管理者」ロールチップを表示

### ダーク/ライトモード
全画面共通。ヘッダー右上のトグルボタンで切替。  
`localStorage` に保存し、ページ間で状態を引き継ぐ。
